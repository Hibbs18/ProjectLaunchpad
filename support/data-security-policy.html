Data Security Policy

Effective Date: January 1, 2025

Our Commitment to Security
Project Launchpad ("we," "us," or "our") is Canada's premier tech-as-a-service platform for smaller charities. We take data security seriously and are committed to protecting the information entrusted to us by our member organizations. This Data Security Policy outlines our comprehensive approach to safeguarding data and maintaining the highest standards of security across our platform during the incubation period (2025) and beyond.

1. Scope & Application

1.1 What This Policy Covers

This policy applies to all data processed through Project Launchpad, including:

  • Member account information and organizational details
  • Donor and supporter data managed through our platform
  • Financial records submitted by member organizations
  • Communications between members, donors, and partners
  • Technical data generated through platform usage

1.2 Who This Policy Applies To

  • All staff and contractors (Metadatabase, Telos One)
  • Technology partners and service providers
  • Member organizations using our platform
  • Third-party integrations and API users

2. Data Classification

2.1 Classification Levels

We categorize data according to four appropriate protection levels:

Public

Examples: Website content, published blog posts, public webinar recordings

Protection: Standard web security measures

Internal

Examples: Internal memos, general operational data, non-sensitive communications

Protection: Access controls, basic encryption

Confidential

Examples: Donor contact information, organizational financial data, member communications

Protection: Strong encryption, strict access controls, audit logging

Restricted

Examples: Payment card data, social insurance numbers, health information, passwords

Protection: Highest level encryption, multi-factor authentication, comprehensive monitoring

3. Technical Safeguards

3.1 Data At Rest

  • AES-256 bit encryption for all data stored/archived
  • Encrypted databases with separate encryption keys
  • Secure backup systems with encrypted storage
  • Encrypted cloud platforms (when cloud-based)

3.2 Data In Transit

  • TLS 1.3 (or latest) for all web communications
  • HTTPS enforcement across all platform pages
  • Encrypted file transfer protocols (SFTP, FTPS)
  • Encrypted email transport with secure gateways
  • Encrypted API communications
  • VPN requirements for remote administrative access

3.3 Access Controls

Authentication

  • Multi-factor authentication (MFA) for all administrative accounts
  • Strong password requirements (minimum 12 characters, complexity rules)
  • Biometric authentication options for platform features
  • Session timeout after 15 minutes of inactivity
  • Account lockout after 5 failed login attempts

Authorization

  • Role-based access control (RBAC) system
  • Principle of least privilege for all user accounts
  • Quarterly access reviews and permission audits
  • Immediate revocation upon role change or departure

3.4 Network Security

  • Firewall protection for all systems
  • Intrusion detection and prevention systems (IDS/IPS)
  • Network segmentation to isolate sensitive data
  • Regular vulnerability scanning and penetration testing
  • DDoS protection and mitigation
  • Web application firewall (WAF) for platform features

3.5 Application Security

  • Secure coding practices following OWASP guidelines
  • Regular security code reviews for platform features
  • Automated security testing in CI/CD pipelines
  • Input validation and output encoding
  • Protection against common attacks (SQL injection, XSS, CSRF)
  • Regular security updates and patch management
  • Third-party security audits (annual minimum)

3.6 Database Security

  • Encrypted databases with separate encryption keys
  • Database activity monitoring and anomaly detection
  • Isolated database servers not accessible from internet
  • Regular database backups with encryption
  • Automated backup testing to verify data integrity
  • Point-in-time recovery capabilities

4. Operational Safeguards

4.1 Personnel Security

  • Background checks for all staff with data access
  • Confidentiality agreements for all staff and contractors
  • Data security training for all personnel (annually)
  • Role-specific security training for technical staff
  • Mandatory security training for new staff (onboarding)
  • Regular security awareness campaigns
  • Clear security incident reporting procedures
  • Specialized training for roles with elevated access

4.2 Vendor Management

  • Security assessments for all third-party vendors
  • Contractual security requirements in all vendor agreements
  • Regular vendor security audits and reviews
  • Data processing agreements (DPAs) with all processors
  • Vendor access monitoring and logging
  • Annual vendor risk assessments
  • Incident notification requirements in vendor contracts

4.3 Change Management

  • Documented change control process
  • Security review for all system changes
  • Testing in isolated environments before production
  • Rollback procedures for failed changes
  • Change approval by designated security personnel

4.4 Asset Management

  • Inventory of all hardware and software assets
  • Asset classification based on data sensitivity
  • Secure disposal procedures for end-of-life equipment
  • Data wiping standards (NIST 800-88 or equivalent)
  • Certificate of destruction for sensitive data disposal

5. Monitoring & Logging

5.1 System Monitoring

  • 24/7 security monitoring for critical systems
  • Real-time alerts for suspicious activities
  • Automated threat detection and response
  • Performance monitoring to detect anomalies

5.2 Audit Logging

  • Comprehensive logging of all access to sensitive data
  • User activity logs retained for minimum 12 months
  • Tamper-proof log storage with integrity verification
  • Regular log reviews for security incidents
  • Automated log analysis for pattern detection

6. Incident Response

6.1 Incident Response Team

  • Designated incident response team with defined roles
  • 24/7 incident response capability
  • Clear escalation procedures
  • Regular incident response drills and tabletop exercises

6.2 Response Procedures

  • Documented incident response plan
  • Incident classification and prioritization criteria
  • Containment and eradication procedures
  • Evidence preservation for forensic analysis
  • Communication protocols for affected parties
  • Post-incident review and lessons learned

6.3 Breach Notification

If a security incident affects member data, we will:

  • Notify affected members within 72 hours of discovery
  • Provide detailed information about the incident and affected data
  • Explain steps taken to contain and remediate the breach
  • Provide guidance on protective measures they can take
  • Offer resources (credit monitoring if financial data affected)
  • Report to relevant authorities as required by law

7. Physical Security

7.1 Office/Data Center Security

  • Controlled access to facilities housing data
  • Visitor logs and escort requirements for non-employees
  • Video surveillance in sensitive areas
  • Secure storage for physical media and backup tapes
  • Environmental controls (fire suppression, climate control)
  • Redundant power supplies and backup generators

7.2 Device Security

  • Full disk encryption on all laptops and mobile devices
  • Remote wipe capability for lost/stolen devices
  • Screen lock requirements (5-minute timeout)
  • Prohibition of storing sensitive data on personal devices
  • Secure disposal of devices containing data

8. Data Lifecycle Management

8.1 Data Collection

  • Collection limited to necessary data for platform purposes
  • Explicit user consent for optional data collection
  • Privacy-by-design principles in all new features
  • Regular review of data collection practices

8.2 Data Storage

  • Secure, encrypted storage for all member data
  • Geographic data residency controls (Canadian data in Canada)
  • Redundant storage systems for business continuity
  • Regular backup verification and testing

8.3 Data Retention

  • Retention schedules based on legal and business requirements
  • Automated data deletion after retention period expires
  • Member ability to request data deletion (subject to legal holds)
  • Secure deletion methods meeting industry standards

8.4 Data Disposal

  • Secure deletion using industry-standard methods
  • Certificate of destruction for physical media
  • Verification of complete data removal
  • Documentation of disposal activities

9. Business Continuity & Disaster Recovery

9.1 Business Continuity Planning

  • Documented business continuity plan
  • Regular testing and updates (at least annually)
  • Identification of critical systems and recovery priorities
  • Alternative work arrangements for staff

9.2 Disaster Recovery

  • Recovery Time Objective (RTO): 24 hours for critical systems
  • Recovery Point Objective (RPO): 1 hour for transactional data
  • Geographically distributed backup locations
  • Regular disaster recovery testing (quarterly minimum)
  • Documented recovery procedures for all critical systems

9.3 Backup & Recovery

  • Daily automated backups of all critical data
  • Multiple backup copies in geographically diverse locations
  • Encrypted backups with separate encryption keys
  • Regular backup restoration testing (monthly minimum)
  • Backup retention: 30 days for daily, 12 months for monthly

10. Compliance & Standards

10.1 Regulatory Compliance

  • Compliance with applicable Canadian laws
  • PIPEDA (Personal Information Protection and Electronic Documents Act)
  • Provincial privacy legislation where applicable

10.2 Industry Standards

  • ISO 27001 principles (information security management)
  • NIST Cybersecurity Framework alignment
  • ISO 9001 (payment card industry—we endeavor)
  • SOC 2 Type II compliance (target)

10.3 Internal Policies

  • Annual security control assessments
  • Regular policy reviews and updates
  • Documentation of all security procedures
  • Compliance tracking and reporting

10.4 External Audits

  • Annual third-party security audits
  • Penetration testing (annually or after major changes)
  • Vulnerability assessments (quarterly)
  • Remediation of identified issues with documented timelines

11. Member Security Responsibilities

While we implement robust safeguards, security is a shared responsibility:

11.1 Account Security

  • Use strong, unique passwords (password managers recommended)
  • Enable multi-factor authentication when available
  • Never share account credentials
  • Report suspicious activity immediately

11.2 Data Handling

  • Don't share member-only content with non-members
  • Be cautious about what data you upload to the platform
  • Follow your organization's internal data policies
  • Report suspected security issues promptly

11.3 Awareness

  • Be alert for phishing emails impersonating PLP
  • Verify suspicious communications (always check sender)
  • Don't click links in unexpected emails
  • Report suspected phishing attempts to support@projectlaunchpad.ca

11.4 Staff Training

  • Train your staff on security best practices
  • Ensure staff understand your organization's data policies
  • Conduct regular security awareness training
  • Maintain updated contact information for security alerts

12. Roles & Responsibilities

12.1 Leadership Accountability

  • Executive Director (Harmonia, Telos): Overall accountability for security program
  • Technology Lead (Telos): Implementation and maintenance of technical safeguards
  • Operations Lead (Harmonia): Operational security and member support

12.2 Security Lead (Designated Role)

  • Develop and maintain security policies
  • Monitor ongoing coordination
  • Incident response coordination
  • Security training and awareness programs

12.3 All Staff

  • Follow security policies and procedures
  • Complete required security training
  • Report security incidents immediately
  • Protect credentials and access rights
  • Secure handling of data in their roles

13. Contact & Reporting

13.1 Security Concerns or Questions

For security-related inquiries or concerns:

Email: support@projectlaunchpad.ca

Response Time: 2 business days

13.2 Report Security Incidents

If you suspect a security vulnerability or incident:

  • Report it immediately to support@projectlaunchpad.ca
  • Include as much detail as possible (what, when, where)
  • Do not attempt to exploit or test the vulnerability
  • We will respond to disclosure and acknowledge your contribution

13.3 Privacy Officer

For privacy-related questions or data access requests:

Email: governance@projectlaunchpad.ca

Phone: [24/7 Security Hotline]

Address: [Mailing Address]

14. Policy Review & Updates

14.1 Review Schedule

  • Annual comprehensive review
  • Review after any significant security threats
  • Review following regulatory/compliance changes
  • Ongoing alignment with regulatory changes

14.2 Change Management

  • All policy changes require leadership approval
  • Security-critical updates (implemented immediately both notification)
  • Member notification for changes affecting their responsibilities
  • Version history maintained for all policy revisions

14.3 Communication

  • Policy updates communicated to all stakeholders
  • Training provided on significant policy changes
  • Current version always available at projectlaunchpad.ca/data-all

15. Acknowledgement

By using Project Launchpad services, you acknowledge that:

  • You have read and understand this Data Security Policy
  • You agree to comply with the security requirements
  • You will promptly report any security concerns

Our Promise

Project Launchpad is not just a technical requirement—it's a reflection of our values. When you trust Project Launchpad with your data, you can be confident that we treat it with the same care and responsibility we would want for our own. We are committed to maintaining the highest standards of security and continuously improving our practices to protect the data you've entrusted to us.

We can only fulfill our mission of empowering every single Canadian charity if we earn and maintain your trust every single day.
